This guide gets GenAI telemetry flowing into Parseable in minutes. It covers three instrumentation paths for capturing LLM call data. Once data is flowing, the [Instrumentation Guide](/docs/user-guide/agent-observability/instrumentation-guide) shows how to add full agent-level instrumentation: `invoke_agent` spans for agent runs, `execute_tool` spans for tool executions, problem statements, and completion summaries.

Choose the path that fits your stack, then configure either an OTel Collector or direct export.

## Instrumentation Paths

### Path A: Zero-Code Auto-Instrumentation (OpenAI SDK)

If you are using the OpenAI Python SDK, the official OpenTelemetry distro can instrument all LLM calls with zero code changes.

**1. Install dependencies**

```bash
pip install opentelemetry-distro opentelemetry-exporter-otlp
opentelemetry-bootstrap -a install
```

**2. Set environment variables**

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318
export OTEL_SERVICE_NAME=my-agent
export OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT=true
export OTEL_TRACES_EXPORTER=otlp
export OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf
```

**3. Run your application with the auto-instrumentation wrapper**

```bash
opentelemetry-instrument python my_agent.py
```

That is it. Every `openai.chat.completions.create()` call will emit `gen_ai.*` spans automatically.

---

### Path B: Two-Line Auto-Instrumentation (Any LLM Provider)

For broader provider support (Anthropic, Cohere, Mistral, Bedrock, VertexAI, and more), use OpenLLMetry (Traceloop SDK) or OpenLIT. These require only two lines of code added to your application entry point.

**Option 1: OpenLLMetry (Traceloop SDK)**

```bash
pip install traceloop-sdk
```

```python
from traceloop.sdk import Traceloop

Traceloop.init()  # Call once at application startup
```

**Option 2: OpenLIT**

```bash
pip install openlit
```

```python
import openlit

openlit.init()  # Call once at application startup
```

Both libraries auto-patch supported LLM client libraries and emit standard `gen_ai.*` OTel spans. Configure the OTLP exporter endpoint using the same environment variables shown in Path A.

---

### Path C: Manual Instrumentation (Full OTel SDK)

For maximum control, you can manually create spans with `gen_ai.*` attributes using the OpenTelemetry SDK directly.

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.semconv._incubating.attributes import gen_ai_attributes as GenAIAttributes

# Setup
provider = TracerProvider()
exporter = OTLPSpanExporter(endpoint="http://localhost:4318/v1/traces")
provider.add_span_processor(BatchSpanProcessor(exporter))
trace.set_tracer_provider(provider)
tracer = trace.get_tracer("my-agent")

# Instrument an LLM call
with tracer.start_as_current_span("chat gpt-4o") as span:
    span.set_attribute(GenAIAttributes.GEN_AI_OPERATION_NAME, "chat")
    span.set_attribute(GenAIAttributes.GEN_AI_SYSTEM, "openai")
    span.set_attribute(GenAIAttributes.GEN_AI_REQUEST_MODEL, "gpt-4o")
    span.set_attribute(GenAIAttributes.GEN_AI_REQUEST_TEMPERATURE, 0.7)
    span.set_attribute(GenAIAttributes.GEN_AI_REQUEST_MAX_TOKENS, 1024)

    # ... make your LLM call here ...

    span.set_attribute(GenAIAttributes.GEN_AI_RESPONSE_MODEL, "gpt-4o-2024-08-06")
    span.set_attribute(GenAIAttributes.GEN_AI_RESPONSE_ID, "chatcmpl-abc123")
    span.set_attribute(GenAIAttributes.GEN_AI_RESPONSE_FINISH_REASONS, ["stop"])
    span.set_attribute(GenAIAttributes.GEN_AI_USAGE_INPUT_TOKENS, 150)
    span.set_attribute(GenAIAttributes.GEN_AI_USAGE_OUTPUT_TOKENS, 42)
```

## Collector Configuration

### Option 1: OTel Collector (Recommended)

Deploy an OpenTelemetry Collector between your application and Parseable for buffering, batching, and reliability. Save the following as `parseable-genai-collector.yaml`:

```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
      http:
        endpoint: 0.0.0.0:4318

processors:
  batch:
    timeout: 5s
    send_batch_size: 256

exporters:
  otlphttp/parseable:
    endpoint: ${PARSEABLE_URL}
    encoding: json
    headers:
      Authorization: "Basic ${PARSEABLE_AUTH}"
      X-P-Stream: "${STREAM_NAME}"
      X-P-Log-Source: "otel-traces"
      X-P-Dataset-Tag: "agent-observability"

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlphttp/parseable]
```

Run the collector:

```bash
export PARSEABLE_URL=http://parseable-host:8000
export PARSEABLE_AUTH=dXNlcjpwYXNz          # base64(username:password)
export STREAM_NAME=genai-traces

otelcol-contrib --config parseable-genai-collector.yaml
```

### Option 2: Direct-to-Parseable (No Collector)

For simpler setups, you can export traces directly from your application to Parseable by setting OTLP environment variables. No collector process is needed.

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT=http://parseable-host:8000
export OTEL_EXPORTER_OTLP_HEADERS="Authorization=Basic dXNlcjpwYXNz,X-P-Stream=genai-traces,X-P-Log-Source=otel-traces,X-P-Dataset-Tag=agent-observability"
export OTEL_SERVICE_NAME=my-agent
export OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT=true
```

Then run your instrumented application (Path A, B, or C) as normal. Traces will be sent to Parseable's `/v1/traces` endpoint.

## Verify Data Ingestion

Once your application is running, verify that trace data is arriving in Parseable:

```sql
SELECT COUNT(*) AS span_count,
       MIN(p_timestamp) AS first_seen,
       MAX(p_timestamp) AS last_seen
FROM "genai-traces"
WHERE p_timestamp > NOW() - INTERVAL '1 hour';
```

If `span_count` is greater than zero, your LLM call instrumentation is working. Next, follow the [Instrumentation Guide](/docs/user-guide/agent-observability/instrumentation-guide) to add agent-level spans (`invoke_agent`, `execute_tool`) for full end-to-end agent observability. You can also explore the [Schema Reference](/docs/user-guide/agent-observability/schema-reference) and [SQL Query Templates](/docs/user-guide/agent-observability/sql-queries) to start querying your data.
