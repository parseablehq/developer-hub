This section covers the main HTTP endpoints that developers use with Parseable. It includes the older JSON and dataset APIs, along with the newer native OTLP endpoints for logs, traces, metrics, and Prometheus remote write.

## Getting Started

Before you begin, make sure you have:
- A running instance of Parseable
- The base URL of your Parseable server
- Required authentication credentials (if authentication is enabled)

Most integrations fall into one of these paths:

- Use `/api/v1/ingest` when you want to send JSON events and select the destination dataset with headers.
- Use `/v1/logs`, `/v1/traces`, or `/v1/metrics` when you are sending native OTLP data.
- Use `/v1/prometheus/write` when you are forwarding Prometheus Remote Write traffic.
- Use `/api/v1/query` and `/api/v1/logstream/{stream_name}` for SQL querying and dataset management.

## Authentication

API reference endpoints require authentication. The OpenAPI reference uses HTTP Basic authentication by default:

```bash
Authorization: Basic <base64-encoded-credentials>
```

If you use Parseable API keys, send the key in the `X-API-Key` header as described in the [API Keys](/docs/user-guide/api-keys) guide.
